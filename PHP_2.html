自动加载

源自：https://www.zybuluo.com/phper/note/66447

1. 自动加载原理及__autoload的使用

主要是实现类的自动加载，不再需要逐个文件的引入操作。可以看到下方，new class的时候会自动调用__autoload方法

index.php

define('ROOT_PATH', dirname(_file_) . '/');
define('CORE_PATH', ROOT_PATH . 'core/');

// 创建对象
$db = new DB();

// 静态方法直接调用
Info::test();

function __autoload($className)
{
    require CORE_PATH . $className . '.class.php';
}

./core/DB.class.php

class Db
{
    
    function __construct()
    {
        echo 'Hello DB';
    }
}

2. spl_autoload_register的使用

小项目__autoload方法基本能实现，如果类在多个文件夹，就需要spl_autoload_register
执行效率更高，更灵活

define('ROOT_PATH', dirname(__FILE__) . '/');
define('CORE_PATH', ROOT_PATH . 'core/');
define('LIB_PATH',  ROOT_PATH . 'lib/');

function load1($className)
{
    echo 'core: ';
    include_once CORE_PATH . $className . '.class.php';
    
}

function load2($className)
{
    echo 'lib: ';
    include_once LIB_PATH . $className . '.class.php';
}

spl_autoload_register('load1');
spl_autoload_register('load2');
// spl_autoload_register使用时，__autoload会无效，有时候，我们希望它继续有效，就可以也将它注册进来，就可以继续使用。
spl_autoload_register('__autoload');

// core 目录创建对象
$db = new DB();

// 静态方法直接调用
Info::test();

// lib 目录创建对象
$redis = new Redis();

分析如下：
1. new DB的时候，就按照注册顺序，先去找load1()函数了，发现找到了，就停止了
2. new Info的时候，先是安装注册顺序，先找load1(), 发现没找到，就去load2()里面去找，还是没这个文件，就去__autoload()函数里找，再输出Hello Info

Redis类

/**

namespace Lib;

{

    {

    }

自动加载类
AutoLoading\loading

<?php

namespace AutoLoading;

class loading
{
    public static function autoload($className)
    {

        $fileName = str_replace('\\', DIRECTORY_SEPARATOR, DIR . '\\'. $className . '.class.php');

        if (is_file($fileName)) {
            include_once $fileName;
        } else {
            echo $fileName . ' is not exist '; die;
        }
    }
}

主函数
index.php

define('DIR', dirname(__FILE__));

require DIR . '/loading.php';


spl_autoload_register("\\AutoLoading\\loading::autoload");


Lib\Redis::test();
