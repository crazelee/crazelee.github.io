Redis的数据结构（String）

http://redisbook.com/preview/object/string.html

一、数据结构

简单动态字符串 simple dynamic string， SDS

struct sdshdr {
    // 记录 buf 数组中已使用字节的数量
    // 等于 SDS 所保存字符串的长度
    int len;
    // 记录 buf 数组中未使用字节的数量
    int free;
    // 字节数组，用于保存字符串
    char buf[];

};

数据结构有点类似PHP里的字符串结构，存有一个len字段，用于获取字符串的长度

二、Redis key规则

  • KEY不要太长，不仅消耗内存，而且在数据中查找的计算成本很高；
  • 使用一种模式，“object-type:id:field”，例如：“user:1000:password”

三、字符串（string）

使用该类型，Redis相当于可持久化的memcached服务器
（注：mem数据仅保存在内存中，服务器重启后数据将丢失）

普通操作

> set user:name crazelee  // 设置
OK

> get user:name   // 获取
"crazelee"

> del user:name   // 删除
(integer) 1

生命周期

> set user:name crazelee ex 10  // 设置10秒过期
OK

> ttl user:name  // 查看生命期
(integer) 3

> expire user:name // 设置过期时间

进阶操作

> getset user:100:name lee  // 获取旧值并重新赋值
"crazelee"

> get user:100:name
"lee"

> mset user:100:name crazelee user:100:score 96 // 设置多个key
OK

> mget user:100:name user:100:score
1) "crazelee"
2) "96"

修改操作

> strlen user:100:name // 获取
> append user:100:name world  // 拼接

INCR key：原子操作加1。

  • key不存在，默认值为0，再加1;
  • key存在，但是存储的不能表示为整数，执行该操作会报错；

应用场景
1. 验证码，通常有过期时间，一般10分钟，那就直接设置过期时间；
