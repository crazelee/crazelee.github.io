Redis的数据结构（List）

一、数据结构

双向链表

二、链表（list）

Redis的list就是一个极其强大的队列系统。

基本操作: PUSH/POP

> rpush queue a  // 从右边进队列
(integer) 1
> rpush queue b
(integer) 2
> lrange queue 0 -1  // 获取队列内容
1) "a"
2) "b"
> lpop queue  // 从左边出队列
"a"

进阶操作

> rpoplpush source_list dest_list

原子性地返回并移除存储在 source 的列表的最后一个元素（列表尾部元素）， 并把该元素放入存储在 destination 的列表的第一个元素位置（列表头部）。

应用场景1：安全的队列。例如，当接收到消息后，出现了网络问题或者消费者端崩溃了， 那么这个消息就丢失了。问题：消费者端取到消息的同时把该消息放入一个正在处理中的列表。 当消息被处理了之后，该命令会使用 LREM 命令来移除正在处理中列表中的对应消息。
另外，可以添加一个客户端来监控这个正在处理中列表，如果有某些消息已经在这个列表中存在很长时间了（即超过一定的处理时限）， 那么这个客户端会把这些超时消息重新加入到队列中。

应用场景2：可以作为补发队列。
