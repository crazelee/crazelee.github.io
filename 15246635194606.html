RDB持久化

Redis数据存储在内存里，为了持久化，会将数据库保存在磁盘中。

持久化可以手动执行，也可配置定期执行。

一、 RDB文件的创建与载入

两个REDIS命令可以生成RDB文件，一个是SAVE，一个是BGSAVE。

  • SAVE：阻塞Redis服务器进程，直到RDB文件创建完毕，服务器不能处理任何命令请求。
  • BGSAVE：派生出子进程，由子进程创建RDB文件，服务器进程（父进程）继续处理其他命令请求。

伪代码

def SAVE():
    # 创建RDB文件
    redbSave()
    
def BGSAVE():
    #创建子进程
    pid = fork()
    if pid == 0
        #子进程负责创建RDB文件
        rdbSave()
        #完成之后向父进程发送信号
        signal_parent()
    elif pid > 0: 
        #父进程继续处理其他命令请求，并通过轮询等待子进程的信号
        handle_request_and_wait_signal()
    else
        #处理出错情况
        handle_fork_error

载入RDB文件过程：服务器启动时自动检测AOF文件、RDB文件，检测到自动执行。载入的过程中处于阻塞状态。

二、 自动间隔性保存

服务器配置
# 服务器在900秒内，对数据库进行了至少1次修改
save 900 1
# 服务器在300秒内，对数据库进行了至少10次修改
save 300 10

struct redisServer {
    //...
    // 记录保存条件的数组
    struct saveparam *saveparams;
    
    // 修改计数器
    long long dirty;
    
    // 上一次执行保存的时间
    time_t lastsave;
}

struct saveparam {
    // 秒数
    time_t seconds;
    
    // 修改数
    int changes;
}

说明：
当服务器成功执行一次数据库修改命令之后，dirty计数器就增加一次

定期检查策略

周期性操作函数serverCron默认每隔100毫秒执行一次，其中一项是检查save选项所配置的保存条件

伪代码

def serverCron():
    # ...
    # 遍历所有保存条件
    for saveparam in server.saveparams:
    
        # 计算距离上次执行保存操作有多少秒
        save_interval = unixtime_now() - server.lastsave
        
        # 计数器大于配置 && 间隔时间大于配置
        if server.dirty >= saveparam.changes and save_interval > saveparam.seconds: 
            BGSAVE()
            
        # 执行BGSAVE后，dirty便置为0，lastsave也变为执行操作时间

RDB文件是一个经过压缩的二进制文件，由多个部分组成
